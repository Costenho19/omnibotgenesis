**Three-Tier Mandate Certification (MIVP — ADR-194):**
- **MANDATE-BOUND** (Tier 1 — pristine): `turns_in_violation = 0` AND `turns_in_warning = 0`. Highest mandate fidelity. Zero detected drift signals throughout session. Governed by MIVP-INV-008.
- **MANDATE-ALIGNED** (Tier 2 — mission-aligned): `turns_in_violation = 0` AND `turns_in_warning > 0`. Agent never crossed the HALT threshold; warning-level drift occurred. Governed by MIVP-INV-009.
- **UNCERTIFIED** (Tier 3): `turns_in_violation > 0`. Violations recorded — both higher tiers withheld.
- MANDATE-BOUND and MANDATE-ALIGNED are **mutually exclusive** — only one tag is ever appended to a PoGC. Enforced by DB CHECK constraint `chk_seal_tier_consistency`.

**The Mandate vs. Constraint Distinction (critical):**
- A **constraint** defines what the agent MUST NOT do — a negative bound on the action space (BEV/CCS layer, ADR-182).
- A **mandate** defines what the agent MUST optimize for — a positive objective the agent is directed toward (MIVP layer, ADR-194).
- These are orthogonal: an agent can satisfy all constraints (CCS: CONFORMANT) while violating its mandate (MAS: WARNING/HALT). This is the proxy-optimization failure mode MIVP closes.
