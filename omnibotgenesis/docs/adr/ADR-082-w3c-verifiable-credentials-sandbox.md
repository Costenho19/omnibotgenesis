# ADR-082: W3C Verifiable Credentials — Public Governance Sandbox

**Status:** ACCEPTED  
**Date:** April 2026  
**Author:** Harold Nunes, OMNIX QUANTUM LTD  
**Scope:** `omnix_web/api/sandbox.py` · `omnix_web/api/server.py`

---

## Context

OMNIX governance receipts are PQC-signed JSON objects. After every sandbox evaluation
(`POST /api/sandbox/evaluate`), the receipt was returned in a proprietary OMNIX format.
External systems — DID resolvers, EUDI wallet software, institutional compliance tools —
cannot parse a proprietary format without OMNIX-specific integration work.

The W3C Verifiable Credentials Data Model (VCDM 1.1) is the open standard for
machine-verifiable claims. Wrapping OMNIX receipts in the VC envelope makes them
interoperable with any W3C VC-aware system without requiring OMNIX-specific code.

---

## Decision

Every governance decision generated by the public sandbox is also returned as a
W3C Verifiable Credential alongside the native OMNIX receipt.

### VC structure

```json
{
  "@context": [
    "https://www.w3.org/2018/credentials/v1",
    "https://omnixquantum.net/schemas/omnix-receipt-v1.jsonld"
  ],
  "id": "https://did.omnixquantum.net/receipts/{receipt_id}",
  "type": ["VerifiableCredential", "GovernanceDecisionCredential"],
  "issuer": {
    "id": "https://did.omnixquantum.net",
    "name": "OMNIX Quantum Ltd"
  },
  "issuanceDate": "{ISO-8601 timestamp}",
  "credentialSubject": {
    "id": "https://did.omnixquantum.net/receipts/{receipt_id}",
    "receiptId": "{receipt_id}",
    "decision": "APPROVED | BLOCKED | HOLD",
    "asset": "{asset}",
    "domain": "{domain}",
    "checkpointsPassed": 11,
    "checkpointsTotal": 11,
    "policyVersion": "{policy_version}",
    "contentHash": "{sha256_hex}",
    "verificationUrl": "https://omnixquantum.net/verify/{receipt_id}"
  },
  "proof": {
    "type": "Dilithium3Signature2024",
    "created": "{timestamp}",
    "verificationMethod": "https://did.omnixquantum.net/keys/{key_id}",
    "signatureAlgorithm": "Dilithium-3",
    "proofValue": "{base64_signature}"
  }
}
```

### Proof type mapping

| Signature algorithm | `proof.type` |
|---------------------|-------------|
| Dilithium-3 (PQC) | `Dilithium3Signature2024` |
| Any other | `SHA256HashChain2024` |

## Implementation

**File:** `omnix_web/api/sandbox.py`  
**Function:** `_build_w3c_vc()` (lines ~140–189)

- `OMNIX_ISSUER_URL = "https://did.omnixquantum.net"` — DID issuer
- `VERIFICATION_BASE_URL = "https://omnixquantum.net/verify"` — verification endpoint
- `OMNIX_VC_CONTEXT` — JSON-LD context list (W3C standard + OMNIX extension)

The sandbox `evaluate()` endpoint attaches the VC under the `verifiable_credential` key
in the JSON response. The native OMNIX receipt is preserved under `receipt`.

## Interoperability targets

- **EUDI Wallet** (EU Digital Identity) — consumes W3C VC natively
- **DID resolvers** — can resolve `did:web:omnixquantum.net` issuer
- **Institutional compliance tools** — any W3C VC-aware system
- **Velos Capital** — webhook payloads include VC format (ADR-084 endpoint)
